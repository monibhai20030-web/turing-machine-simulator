📼 Tape 1 — Input Tape

Holds the original input string. The read head scans left-to-right during the copy phase, then reverses for palindrome verification. This tape is read-only in the check phase.

📼 Tape 2 — Work Tape

Acts as scratch space. A copy of the input is written here, then read backwards during the palindrome check. This allows simultaneous bidirectional comparison impossible on a single tape.

⚡ Efficiency Gain

A single-tape machine needs O(n²) steps for palindrome checking. The 2-tape machine does it in O(n) steps — a polynomial speedup that demonstrates the power of multi-tape computation.

🔄 Transition Function

δ(q, a, b) → (q', a', b', D₁, D₂) — reads symbols from both tapes simultaneously, writes to both, and moves each head independently. Three phases: Copy → Rewind → Check.

SINGLE-TAPE TIME

n=5~25 steps
n=10~100 steps
n=20~400 steps

Complexity: O(n²) — the head must sweep back and forth across the full tape for each symbol comparison.

2-TAPE TIME

n=5~15 steps
n=10~30 steps
n=20~60 steps

Complexity: O(n) — each head moves once per character. Copy forward, then compare while moving opposite directions.

Theorem (Linear Speedup): Any k-tape Turing machine with time complexity T(n) can be simulated by a single-tape machine in time O(T(n)²). Conversely, a 2-tape machine can solve palindrome in O(n), while any single-tape machine requires Ω(n log n) steps. This proves the multi-tape model is strictly more efficient (though not more powerful — both recognize exactly the class of Turing-recognizable languages).

Single-Tape vs Multi-Tape — Key Differences

📼 Single-Tape Machine

⚠Must physically move the head back to the start after reading the input — wastes O(n) steps per comparison.
⚠Palindrome check requires repeated back-and-forth sweeps, giving O(n²) total time.
⚠State must encode position information implicitly — transition table grows larger.
✓Simpler hardware model. Easier to reason about theoretically.

📼📼 Multi-Tape Machine

✓Copy phase duplicates input in O(n) steps. Both heads then move simultaneously in opposite directions.
✓Check phase runs in O(n) — each step compares one character from each end simultaneously.
✓Transition function reads and writes multiple symbols per step — more expressive per computation step.
✓Equivalent power to single-tape — same languages recognized — but polynomially faster.
